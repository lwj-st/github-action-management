一、 产品概述
1.1 背景
GitHub Actions 已成为 CI/CD 与自动化任务的核心工具，但官方 Web 界面在多仓库、多账户管理、参数化触发、结果聚合与实时控制方面存在操作碎片化、交互效率低等问题。开发者/DevOps 工程师亟需一款本地化、跨平台、高颜值、可集中管控的桌面客户端。
1.2 产品定位
一款面向开发者与运维团队的轻量级 GitHub Actions 本地管控桌面应用，支持多账户仓库检索、参数化触发、实时状态监控、结果提取与流程干预，兼顾安全性、实用性与现代 UI 体验。
1.3 目标用户
独立开发者 / 开源维护者
前端/后端/测试工程师
DevOps / SRE 团队
技术管理者（需批量监控与结果归档）
二、 核心功能需求
2.1 多账户与仓库管理
需求项
详细说明
验收标准
多账户接入
支持添加多个 GitHub 账号（PAT 或 OAuth），独立隔离配置
可并行管理 ≥5 个账户，切换无延迟，凭据不串扰
仓库检索
自动拉取账户下有权限的仓库（公开/私有），支持名称、Owner、语言、更新时间筛选
搜索响应 ≤1s（缓存后），支持分页/虚拟列表
配置持久化
账户 Token、常用仓库、参数预设本地加密存储
重启后状态完整恢复，Token 不落盘明文
2.2 Workflow 触发与参数配置
需求项
详细说明
验收标准
自动解析参数
读取仓库 .github/workflows/*.yml，自动识别 workflow_dispatch 的 inputs（string/choice/boolean/environment/number）
动态生成表单，与 GitHub Web 端字段类型、必填校验、默认值完全一致
参数模板
支持保存/加载常用参数组合（预设）
可一键复用历史配置，支持导出/导入 JSON
触发控制
支持指定 branch、ref 或 commit SHA 触发（若 workflow 支持）
触发请求符合 GitHub API 规范，失败返回明确错误码
2.3 执行状态监控
需求项
详细说明
验收标准
状态同步
通过 GraphQL 订阅 + 轮询混合机制获取 queued/in_progress/completed 等状态
状态延迟 ≤3s，断线自动重连
状态可视化
颜色/图标区分：✅成功 ❌失败 🟡运行中 ⏸️已取消 🔵跳过 🟠超时
支持按状态/时间/仓库过滤列表
日志查看
实时流式拉取 Job/Step 日志，支持关键词过滤、高亮、折叠、复制
日志加载 ≤2s，支持 ANSI 色彩渲染
2.4 结果获取与制品下载
需求项
详细说明
验收标准
Summary/输出提取
解析 GITHUB_STEP_SUMMARY、outputs、set-output 等，支持文本/URL/JSON 提取
可配置正则或 JSONPath 抓取关键信息，一键复制
制品下载
自动列出 workflow 产生的 artifacts，支持单文件/批量下载
下载进度可视化，断点续传，默认保存至 ~/Downloads/github-actions-artifacts/
结果归档
支持将本次运行结果（参数、状态、日志片段、制品链接）导出为 Markdown/PDF/JSON
导出文件结构清晰，可直接嵌入报告或 Wiki
2.5 流程控制
需求项
详细说明
验收标准
随时终止
对 in_progress 状态的 workflow 发送取消指令
取消请求 ≤2s 生效，状态同步更新为 cancelled
重新运行
支持全量重跑或从指定失败 Job 重跑
符合 GitHub API rerun 规范，携带原始参数
2.6 测例管理（用户侧 & 产品侧）
注：响应“创建测例，测试要能完全通过”的双重含义
用户侧：Workflow 验证测例
用户可定义“预期结果模板”（如：状态=success、日志包含特定关键字、制品存在某文件），触发后自动比对并生成通过率报告。
产品侧：自动化测试基线
应用内置 CI/CD 测试流水线，要求：
单元测试覆盖率 ≥90%
E2E 测试覆盖核心路径（登录→选仓→填参→触发→监控→取结果→取消）
Mock GitHub API 模拟网络异常/限流/鉴权失败
所有 PR 合并前必须 100% 通过测试门禁
三、 非功能性需求
3.1 安全性
🔐 凭据存储：使用 OS 原生密钥环（macOS Keychain / Windows Credential Manager），绝不写入本地明文文件
🔑 Token 权限最小化：仅申请 repo, workflow, actions 范围，支持 Token 过期提醒与一键轮换
🌐 无云端同步：默认纯本地运行，所有数据仅存于用户设备；可选开启端到端加密的自托管同步服务
📜 操作审计：本地记录触发/取消/下载等操作时间戳，支持导出审计日志
🛡️ 防滥用：触发频率限制、错误指数退避、自动检测恶意 workflow 路径
3.2 实用性 & 性能
💾 智能缓存：仓库列表、YAML 结构、参数默认值本地缓存 24h，减少 API 调用
📦 资源占用：内存常驻 ≤150MB，CPU 空闲 ≤1%，支持后台静默运行
📡 弱网适配：支持离线查看历史运行记录，网络恢复后自动同步
🖱️ 快捷操作：支持全局快捷键（触发、取消、刷新）、系统托盘通知、右键菜单集成
3.3 跨平台兼容性
平台
要求
macOS
支持 Intel & Apple Silicon，适配 Dark/Light 模式，支持 Dock 角标显示任务数
Windows
支持 Win10/11 x64/ARM64，支持任务栏进度通知，适配高 DPI
通用
自动更新（静默/提示模式），崩溃上报（匿名化），多语言基础支持（中/英）
3.4 UI/UX 美观度
🎨 设计系统：基于现代 Design Token（间距/圆角/色彩/阴影），支持暗色/亮色/跟随系统
📊 数据可视化：状态环形图、执行耗时趋势、参数表单卡片化、日志终端风 UI
♿ 无障碍：符合 WCAG 2.1 AA，键盘全导航，屏幕阅读器友好
🧩 布局：左侧账户/仓库树，中部触发面板+运行列表，右侧详情/日志/结果面板，支持拖拽分栏
